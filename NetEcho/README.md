 实现BIO程序开发：同步阻塞IO操作，每一个线程都只会管理一个客户端的连接，这种操作的本质是存在有程序阻塞的问题。<br>
线程的资源总是在不断进行创建，并且所有数据接收里面（Scanner、PrintStream简化了），网络的数据都是长度限制的，传统的数据时需要通过字节数组形式传输<br>
BIO：是需要对数据采用按字节传输<br>
程序问题:性能不高、多线程利用率不高、如果大规模的用户访问，有可能会造成服务器资源耗尽。

NIO提倡使用Buffer代替传统数组操作，减少数组操作复杂度，利用缓存数据的保存与方便的清空操作进行处理。<br>
Reactor模型提倡：公共注册，统一操作，所以会有一系列的Channel

1. BIO(同步阻塞IO): 线程操作，IO实现通讯采用的是阻塞模式
